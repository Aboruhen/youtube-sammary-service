# YouTube Summary Service

Minimal service that takes a YouTube video URL, fetches the transcript, generates a summary, and creates a timed agenda using an LLM (OpenAI or local Ollama).

## Features

- **Fetch transcript** via [youtube-transcript-api](https://github.com/trldvix/youtube-transcript-api) (Java)
- **Fallback** when no captions: chunked processing with **yt-dlp** (audio download) + **OpenAI Whisper API** (speech-to-text)
- **Summary** and **agenda with timestamps** generated by Spring AI (OpenAI or Ollama)
- **Hexagonal architecture** with clear ports and pluggable adapters
- **Maven multi-module** project for easy feature changes

## Tech Stack

- Java 21
- Spring Boot 3.2
- Spring AI (OpenAI / Ollama)
- youtube-transcript-api (Java) for captions

## Build & Run

```bash
# Build all modules (use project settings if your Maven uses a corporate repo that times out)
mvn -s .mvn/settings.xml clean install

# Or without project settings (when Maven Central is reachable):
mvn clean install

# Run the API (set your OpenAI API key)
export OPENAI_API_KEY=sk-...
mvn -pl api spring-boot:run
```

If `mvn clean install` fails due to dependency resolution (e.g. timeout to a corporate mirror), use **`mvn -s .mvn/settings.xml clean install`** so the build uses only Maven Central.

### Using Ollama (local LLM)

1. Install [Ollama](https://ollama.ai) and run e.g. `ollama pull llama3.2`
2. In `api/pom.xml`, replace `spring-ai-open-ai-spring-boot-starter` with `spring-ai-ollama-spring-boot-starter`
3. In `api/src/main/resources/application.yml`, configure Ollama and disable OpenAI (or remove OpenAI dependency)
4. Run the API; no API key needed.

## API

**GET** `/api/summary?videoUrl=https://www.youtube.com/watch?v=VIDEO_ID`  
or  
**GET** `/api/summary?videoId=VIDEO_ID`

**Response:**

```json
{
  "videoId": "dQw4w9WgXcQ",
  "summary": "This video discusses...",
  "agenda": [
    { "timestamp": "0:00", "startSeconds": 0, "title": "Introduction" },
    { "timestamp": "1:30", "startSeconds": 90, "title": "Main topic" }
  ],
  "usedFallback": false
}
```

- `usedFallback`: `true` when transcript was obtained via the fallback provider (chunked processing) instead of YouTube captions.

## Project Structure (Hexagonal + Plugins)

```
youtube-summary-service/
├── domain/                    # Entities, value objects, ports (interfaces)
├── application/               # Use cases (orchestration)

```

## Configuration

| Env / Property | Description |
|----------------|-------------|
| `OPENAI_API_KEY` | Required for **chunked fallback** (Whisper API). Optional if all videos have captions. |
| `server.port` | HTTP port (default 8080) |
| `youtube.summary.whisper.api-key` | Same as `OPENAI_API_KEY`; override if different |
| `youtube.summary.yt-dlp.path` | Path to yt-dlp (default: `yt-dlp`) |
| `youtube.summary.ffmpeg.path` | Path to ffmpeg (default: `ffmpeg`) |

## Fallback: Chunked processing when no captions

When a video has no captions (or none in the requested languages), the service uses the **chunked fallback**:

1. **Download audio** with [yt-dlp](https://github.com/yt-dlp/yt-dlp) (must be on PATH).
2. If the file is under 24 MB, **transcribe** it in one call to **OpenAI Whisper API**.
3. If larger, **split** with **ffmpeg** into ~10‑minute chunks, transcribe each chunk, then merge segments with correct timestamps.

**Requirements:** Install **yt-dlp** and **ffmpeg**, and set **`OPENAI_API_KEY`** (used only for Whisper). If `OPENAI_API_KEY` is not set, the chunked provider is skipped and videos without captions will return 404.

The YouTube provider now tries multiple languages (`en`, `uk`, `de`, `es`, `fr`, `pt`, `ru`), so videos with e.g. Ukrainian-only captions may work without using the chunked fallback.
